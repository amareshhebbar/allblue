# Contributing to AllBlue

We welcome contributions to AllBlue! This project thrives on community collaboration to build safer, faster incident response tools.

## How to Contribute

1.  **Fork the Repository:** Click the "Fork" button at the top of the repository page.
2.  **Clone Your Fork:** Clone your forked repository to your local machine.
    ```bash
    git clone [https://github.com/gvamaresh/allblue.git](https://github.com/gvamaresh/allblue.git)
    ```
3.  **Create a Branch:** Create a new branch for your feature or bugfix.
    ```bash
    git checkout -b feature/your-feature-name
    ```
4.  **Make Changes:** Implement your changes. Ensure any new CLI tool wrappers strictly adhere to the type-safe guidelines and do not permit raw shell execution.
5.  **Commit Your Changes:** ```bash
    git commit -m "feat: description of the new tool wrapper"
    ```
6.  **Push Changes:** Push your changes to your fork.
    ```bash
    git push origin feature/your-feature-name
    ```
7.  **Open a Pull Request:** Go to the original AllBlue repository and open a pull request detailing your additions.

## Code of Conduct

Please adhere to the `CODE_OF_CONDUCT.md` in all interactions. Ensure all discussions on pull requests and issues remain professional and focused on project improvement.
